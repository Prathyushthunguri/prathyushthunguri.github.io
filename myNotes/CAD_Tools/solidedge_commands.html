⚡ Productivity Tips
📐 Part Modelling Best Practices
PracticeWhy
Use Variables for all critical dimensionsDrive design changes from one table — no hunting for dimensions
Name features in PathFinderBOSS_MOUNTING, HOLE_M10 — readable model tree, easier to edit
Sketch on reference planes, not facesFace-based sketches fail if face is deleted. Plane-based sketches are robust
Fully constrain every sketchUnder-constrained sketches move unexpectedly. Green = fully constrained
Use symmetric constraintsHalf-profile + symmetric = one change updates both sides
Apply fillets and chamfers lastDownstream features respect topology — add dressup after all cut/boss features
Use Part Families for variantsOne master part drives length/width variants — no duplicate files
Save templates (.par/.psm/.asm)Company standard materials, units, title block properties saved in template
🔗 Assembly Best Practices
PracticeWhy
Ground one component per assemblyGrounded part = fixed reference. All others relative to it
Use Mate / Planar Align / Axial AlignKinematic relationships — only constrain DOF you intend to remove
Avoid over-constraining componentsToo many mates = redundancy → assembly solve issues
Use sub-assembliesGroup related parts → performance, reuse, and change management
Assembly Sections for reviewSection through assembly without cutting parts — review internal fit
Simplify heavy assembliesSimplify/Substitute references for large hardware (fasteners, bearings) — saves memory
Check Interference DetectionRun before releasing — catch physical clashes
Use Explode for documentationExploded view in Drawing for assembly/service documentation
